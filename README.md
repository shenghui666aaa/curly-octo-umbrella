智能环境监测系统
本项目基于 嵌入式+传感器 开发，全程使用 Claude Code 辅助需求拆解、代码编写、错误调试与功能迭代开发完成。
 一、项目简介
本系统面向实验室、居家、小型室内空间，实现温湿度、光照、空气质量等环境参数的实时采集、数据显示、阈值超限自动告警，支持数据本地存储与实时查看。
二、技术栈
- 硬件：因仑实验箱，环境传感器
- 开发工具：IAR
- AI开发协作：Claude Code，deepseek
- 编程语言：C语言
三、核心功能
1.  多维度环境数据实时采集
2.  数据LCD/串口终端实时可视化输出
3.  异常环境参数声光告警提醒
4.  定时数据记录与查询
四、运行
1.  使用IAR打开工程总文件，完成芯片型号、编译环境配置
2.  编译整个工程，确认0报错、无警告
3.  程序下载烧录至实验箱硬件
4.  上电启动，即可自动运行监测任务
5.  可通过串口工具查看实时日志与运行状态
五、AI（Claude Code）协作开发记录
1.  需求拆解：运用相关skill将整体环境监测任务拆分为传感器驱动、数据处理、显示、告警4个独立模块
2.  代码编写：编写数据处理逻辑、主业务调度代码
3.  代码优化：精简冗余代码、规范代码结构、提升运行稳定性
4.  远程调试：针对编译报错、硬件适配问题、数据异常问题联合排查修复

六、真实排错记录（必填加分项）
 问题现象
程序编译提示 `RouterEB` 报错，传感器初始化后无法正常读取数据，上电后采集数据全部为0或100。
 排查解决
1.  排查硬件引脚定义，发现IO口配置与实验箱实际硬件不匹配
2.  在Claude Code辅助下逐行核对寄存器配置与时钟初始化代码
3.  修正引脚复用与通信时序延时参数
4.  重新编译烧录，数据采集恢复正常，系统稳定运行
